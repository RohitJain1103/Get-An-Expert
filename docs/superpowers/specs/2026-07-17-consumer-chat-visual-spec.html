Judgement call I want checked

You said to remove that the overview was taken from the CLAUDE.md file. I read that as removing the jargon, not the disclosure, so the chip now reads "A short overview of your project".

That matters because the overview is still sent. Naming the file is developer-speak that means nothing to a consumer, but silently shipping a file we no longer mention would trade a small clarity win for a real honesty cost. If you meant drop it completely, say so and it goes.

Resolved: conflict rule

Most recent edit wins, and the customer always beats the expert. Last write wins in the normal case; if both land together, the customer's version survives. Implementation: the relay stamps each edit with a timestamp and an author, and a customer edit is never silently overwritten by an expert edit it did not see.

Still costs two protocol changes, one per direction, since both inbound schemas are validated. That is the price of the feature, now paid deliberately.

The fourth colour: bronze

Chosen to carry meaning, not decoration. Green is people (chat, brand, the expert). Bronze is your machine: the access you granted, and every action taken on it. Now you can tell a message from a file edit while scrolling, without reading a word.

It is midsesh's own #8C7136, darkened to #7D6530 wherever it carries text: the original is 4.33:1 on cream, which fails for body text, and the darker one reaches 5.20:1.

This also caught a real bug. The "rating needed" marker was terracotta #D98A79, which is 2.50:1 on cream and close to unreadable. It is now rust #A04A3C at 5.55:1. Sage fails as text too (2.07:1), so it stays on borders and fills only, which is where it already was.

Claude-native pass, orange excluded

Matched to the Claude desktop app's design language where it carries the premium feel: generous radii (14px cards, pill chips and buttons), one soft shadow level instead of hard edges, an elevated pill composer with a forest focus ring (the composer is the most Claude-identifiable element on any chat surface), and calm hover states on everything touchable.

What was deliberately NOT taken: Claude's coral #D97757, per your instruction; its typefaces, because Cormorant and Hanken are yours and swapping them would make this look like every other Claude-adjacent tool; and its centered narrow column, because a support chat is not a document.

Fonts: the Claude-exact decision, stated honestly

The consumer UI now uses Claude desktop's type system: Styrene B for UI, Copernicus for display, at Claude's scale (16px/1.65 prose, 14px UI, 12px captions). Both are commercial fonts, so their files are not embedded; the stacks name them first and deliver legal stand-ins today: Familjen Grotesk (OFL, the standard Styrene stand-in) and Georgia (Claude's own serif fallback). License Styrene B and Copernicus later and this page picks them up with zero code changes.

The review document around this device keeps the midsesh faces; only the product UI changed.

Motion rules

Three animations total, all under half a second, all killed by prefers-reduced-motion. State changes rise in with a 60ms stagger. The expert card gets the longest, softest entrance on the page, because the claim is the moment the product exists for. The connection dot breathes slowly. Nothing else loops: looping motion in a chat reads as anxiety, not life.

Parked: dark mode

Light only, as agreed. A toggle is cheap to build once and expensive to keep: it doubles the palette to re-check on every future change, and this surface is about to absorb the context card, the expert card, and the waiting page.

Worth knowing it stays easy later. midsesh already runs a dark section (#1C1A16 ground, #F7F3E9 text, sage accent), so a dark theme is derivable from the brand rather than invented. Revisit once the design settles.

What this reverses

The chat page carries a comment today: "palette copied from the dashboard theme on purpose, do not link styles.css". It deliberately matches the expert tool. This change deliberately breaks that, and the comment needs rewriting to say why, or the next person will read it as a mistake and undo it.

It is the right break. The consumer and the expert are not the same user, and one is a customer while the other is an operator.
